The threat landscape shifted the moment LLMs got tool access. Most security tooling hasn't caught up.

I work at that gap — red teaming AI agents, building runtime defenses for LLM systems, and shipping open tools that treat AI security as an engineering problem, not a compliance checkbox.

---

## What I've built

**[AgentGuard](https://github.com/bb1nfosec/Agentguard)** — Runtime security for LLM agents  
Your AI agent has no idea it's been hijacked. AgentGuard does. HMAC-signed tool calls, semantic anomaly detection, prompt injection interception at the boundary — before the model acts.

**[DVAI](https://github.com/bb1nfosec/dvai)** — Deliberately Vulnerable AI Ecosystem  
A full stack of intentionally broken AI systems for training red teamers. Prompt injection, insecure tool use, model inversion — all in one place. The DVWA of AI security.

**[distill](https://github.com/bb1nfosec/Distill)** — Token cost scanner and optimizer  
Scans any codebase and shows exactly which files burn your Claude/GPT tokens and what they cost. Fixes it automatically. CI budget gate + MCP server. `pip install distill-llm`

**[agni](https://github.com/bb1nfosec/agni)** — Open-source EDR simulator  
Written in Rust. Simulates EDR telemetry and detection logic for red teamers who want to test evasion without burning production sensors.

**[chanakya-opsec](https://github.com/bb1nfosec/chanakya-opsec)** — OPSEC failure analysis framework  
Multi-layer signal correlation for attribution analysis. DNS metadata, timing patterns, infrastructure fingerprinting. Research-grade threat modeling.

**[bheeshma](https://github.com/bb1nfosec/bheeshma)** — npm supply chain monitor  
The strace for npm packages. Catches supply-chain attacks at runtime that static analysis misses. Zero dependencies. SARIF output. Ships as a GitHub Action.

**[Vaathi](https://github.com/bb1nfosec/vaathi)** — Free cybersecurity education for India  
Built because quality security training shouldn't cost what it costs. Used by thousands.

**[Information-Security-Tasks](https://github.com/bb1nfosec/Information-Security-Tasks)** ★ 176 — Daily infosec practice  
Community-driven repository of real-world infosec problem statements — forensics, OSINT, malware analysis, red team, blue team. Contributed by practitioners, for practitioners.

## Approach

Security is an adversarial discipline. You don't understand a defense until you've broken it.

I ship tools that work without a sales call, documentation that doesn't require a consultant, and code that can be read in under an hour. Open source by default.
